Congratulations!

You have successfully downloaded and installed DB4Sci! Thanks for downing DB4Sci we hope you find this software helpful. This is a demo install and not indented for production use. DB4Sci should be used with SSL. Your site SSL certificates need to be made part of your local instance. After you completed local configurations for your site this instance can be put into production.

Configure for Production

Use your favorite text editor to complete the configuration of DB4Sci. The configuration file is located at /opt/DB4SCI/mydb/local_config.py. You will need to know how a few things aboutyour local environment.
  • Adminitrative Users - Provide a list of ISIDs for staff who will have admin rights to the application.
  • Active Director - A list of domain controllers and your domain name
  • Email Config - SMTP server, Email alias for users to recieve DB4Sci events; like new container creation and backup notices.
  • Data Volumes - Path to data volumes which will be used to store the database data. More than one volume can be specified. Your site can provide users with multiple options for storage based on performance and cost. Volumes can be filtered by user and database types. Example a volume with limited capacity for administrative users which is separate from a high-performance volume for production databases.
  • Database admin accounts. A list of username and passwords for administrative access. These accounts are independent of the accounts created by users for the purpose of backups.
  • AWS S3 credentials for backups
  • TimeZone - Your local time
