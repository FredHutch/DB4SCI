DB4Sci Design Goals

  • Fast easy way to create database instances. Zero barriers, no request forms, no ticket system.
  • Container Centric. No software installs, just pull Postgres from Docker hub.
  • Get out of the DBA business. Give uses full admin rights to manage thier own databases. But at the same time provide nightly backups.
  • Encourge database useage for scientfic data and work flow management. Databases can be shared and offer current access. Influence the culture and offer better alterative to spreadsheets.
  • DB4Sci is easily extensible. At present MyDB supports four major DB platforms. To add additional platforms just update the configuration file and define the Name, image and version of the DB container. On the backend you need to write Python code to implement a few methods: db_create, db_backup, create_user, check_user.

Depolyment

Docker install is the main requirement for DB4SCI. At the Hutch DB4SCI runs on a dedicated physical server with a few hundred teribyes of nVME SSD storage. The system run Ubuntu 16.04 and storage is configured with ZFS. This configuration is supporting over 50 database instances. DB4SCI could also be deployed in AWS with EC2 instance.
