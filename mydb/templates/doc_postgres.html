Postgres Documentation

Postgres Notes

If you want to tune your database you can change almost all of the configuration settings in postgresql.conf via ALTER SYSTEM commands (e.g., ALTER SYSTEM SET max_connections=100) After making any changes, you must restart the database server through the "Restart DB Container" link. To do so requires a superuser login on that database server.

Users

You can use the following to connect to your database instance
  • DB username and password provided when the postgres instance is created. This user has superuser rights on your database instance. We do not recommend you use your hutchnet id so that the password does not change. Keep your credentials for Postgres databaes in .pgpass file in your home directory. Create .pgpass with permissions of 0600. You can keep credentials for multible databases in the file, one entry per line. Comments are not allowed. The file format for .pgpass is: hostname:port:database:username:password Postgres commands If you do not have psql on your local machine, you can use module load PostgreSQL/9.5.1-foss-2015b to load psql. This loads the entire postgres code base, so other postgres commands (e.g., pg_dump) are also loaded.

    User psql to connect to the database via the following command:

    psql -h mydb -u username -p portnum database

    The port number is provided when you first create the database instance

    You can issue any psql command from here, including create role, grant, create database, alter system. Common commands include create user jdoe;
    create database mybase with owner jdoe;
    grant all privileges on database mybase to myotheruser;
    alter user myotheruser with encrypted password 'MyPassword';
    load psql with database or template: psql template1
    list databases: \l
    delete database: drop database mybase;
    change db owner: alter database mybase owner to myotheruser;
    quit psql: \q

    Create wrapper for easy access

    To make life easier, create a wrapper script, eg psq:

    edit ~/bin/psq and put this content into the file:

    #! /bin/sh PGPASSWORD= psql -h -u -p $@ Change the permisions of the psq file to make it executable.

    chmod +x ~/bin/psq Now you'll find it a lot easier to access your postgres instance, for example you can now simply use this command to list your databases:

    psq --list

    Connecting Microsoft Access to a Postgres database

    You can build a quick front end in MS Access to a postgres database. This is preferable to having your data in Access. To connect Access to Postgres, do the following:
    • Download the appropriate ODBC driver from https://www.postgresql.org/ftp/odbc/versions/. Install the latest version.
    • Open access. Go to External Data > ODBC Database > Link to data source by creating a linked table
    • Enter a DSN name. This is database server and database specific so we recommend a name like mydb_database. Click new.
    • Select PostgreSQL Unicode
    • Save the file in a protected location. Note this will contain your password in clear text.
    • Enter the connection information
      • database=your database name, server=mydb, username=your postgres username, port=docker port number (e.g., 32002), password=your postgres password
